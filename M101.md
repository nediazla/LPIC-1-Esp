# RESUMEN TEÓRICO – EXAMEN LPI 101 (Conceptos y Generalidades)

## 1. Arquitectura del sistema

### Detección y gestión del hardware
Linux detecta el hardware mediante el kernel, que incorpora drivers internos y módulos cargables. El sistema utiliza subsistemas como udev (gestión dinámica de dispositivos) y procfs/sysfs para exponer información del hardware durante el funcionamiento. Los dispositivos de almacenamiento se identifican como bloques (/dev/sdX, /dev/nvmeX) y pueden aparecer dinámicamente mediante hotplug.

### Proceso de arranque
El arranque comienza en BIOS/UEFI, pasa al gestor de arranque (normalmente GRUB2), que carga el kernel y el initramfs.  
El initramfs proporciona un sistema mínimo para montar el sistema de archivos raíz.  
Luego se inicia el proceso init (systemd en la mayoría de distros modernas), que gestiona servicios, dependencias y targets.

### Niveles de ejecución y targets
Los antiguos runlevels SysVinit fueron reemplazados por systemd targets.  
Cada target describe un estado del sistema (multi-user, graphical, rescue…).  
Estos definen qué servicios se inician, detienen o aíslan durante la ejecución.

## 2. Instalación de Linux y gestión de paquetes

### Particionado y sistemas de archivos
Una instalación Linux requiere un esquema de particiones (MBR o GPT).  
Las particiones más comunes incluyen raíz (/), /home, /boot, swap y particiones especiales como EFI.  
Los sistemas de archivos más habituales son ext4, XFS, btrfs, además de pseudo-filesystems como procfs y sysfs.

### Gestor de arranque
GRUB2 es el gestor más extendido. Permite arrancar múltiples sistemas, pasar parámetros al kernel y recuperar sistemas. El archivo principal es `grub.cfg`.

### Librerías compartidas
Linux usa bibliotecas dinámicas (archivos .so) cargadas por el vinculador dinámico.  
La organización sigue rutas configuradas (ej. /lib, /usr/lib), y el sistema mantiene una caché para optimizar la búsqueda.

### Gestión de paquetes (Debian)
El sistema se basa en paquetes .deb administrados mediante dpkg (nivel bajo) y apt (nivel alto).  
APT maneja repositorios, dependencias y actualizaciones del sistema.

### Gestión de paquetes (RPM/YUM/DNF)
Utilizado en sistemas no Debian. RPM gestiona paquetes de bajo nivel, mientras YUM/DNF resuelve dependencias y administra repositorios.

### Virtualización
Linux puede funcionar como anfitrión o huésped.  
Se deben entender conceptos como máquinas virtuales, hipervisores, contenedores y tareas especiales al clonar sistemas (cambios de claves SSH, identificadores, etc.).

## 3. Comandos GNU y UNIX

### Línea de comandos
La CLI permite ejecutar programas, interpretar argumentos y navegar por el sistema. Las herramientas básicas incluyen el uso del path, shells interactivos y documentación integrada mediante man pages.

### Procesamiento de texto
Linux incluye una amplia colección de filtros orientados al procesamiento basado en línea: herramientas para búsqueda (grep), ordenación (sort), comparación, paginación (less) y transformación (cut, tr).

### Gestión de archivos
Incluye operaciones básicas como crear, copiar, mover y eliminar. También incluye inspección de propiedades, enlaces duros y simbólicos, y expresiones globbing para coincidencias de patrones.

### Pipes y redirecciones
El modelo UNIX permite conectar programas entre sí usando pipes. La redirección permite controlar flujos de entrada, salida y error.

### Procesos
Los procesos representan programas en ejecución.  
Tienen estados, señales y prioridades.  
El sistema ofrece herramientas de control, supervisión y finalización.

### Prioridades de procesos
El kernel usa valores de “niceness” para ajustar prioridades. También existen procesos tiempo real.

### Expresiones regulares
Permiten realizar búsquedas avanzadas en archivos usando patrones complejos.

### Edición de texto
Se utilizan editores como vi/vim y nano, además de herramientas de procesamiento como sed y awk.

## 4. Sistemas de archivos y FHS

### Particiones y sistemas de archivos
Linux soporta múltiples tipos de FS, cada uno con ventajas específicas para rendimiento, integridad, snapshots o características avanzadas.

### Integridad del sistema de archivos
Herramientas de chequeo (fsck) detectan y reparan problemas. Los sistemas modernos pueden programar chequeos automáticos tras un número de montajes o periodos de tiempo.

### Montaje de sistemas de archivos
Los FS pueden montarse dinámica o permanentemente mediante fstab. También existen puntos especiales como tmpfs.

### Permisos y propiedad
Linux usa un modelo clásico de permisos (usuario, grupo, otros), complementado con atributos especiales que añaden seguridad adicional.

### Enlaces duros y simbólicos
Son mecanismos para referenciar archivos.  
El hardlink apunta al mismo inode; el symlink apunta a una ruta.

### FHS – Filesystem Hierarchy Standard
Define las ubicaciones correctas de archivos y directorios:
- /etc → configuración  
- /var → datos variables  
- /usr → software del sistema  
- /bin /sbin → programas esenciales  
- /home → datos del usuario  
- /lib → bibliotecas esenciales  

El examen exige reconocer el propósito de cada directorio.

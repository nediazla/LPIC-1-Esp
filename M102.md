## 1. Shells y Shell Scripting

### Personalización del entorno de shell
Las shells en Linux permiten configurar variables, funciones, alias y archivos de inicio para definir el entorno del usuario.  
Los principales archivos de inicio incluyen `/etc/profile`, scripts globales, archivos específicos del usuario (`~/.bash_profile`, `~/.profile`, `~/.bashrc`) y scripts de cierre.  
El shell evalúa estos archivos dependiendo del tipo de sesión: login, no-login, interactiva o no interactiva.  
Los alias y funciones facilitan automatizar tareas repetitivas y estructurar rutinas reutilizables.

### Variables y entorno
El entorno del usuario incluye variables como PATH, MAIL, PS1, LANG o TZ, que influyen en el comportamiento del sistema.  
El modelo distingue entre variables de entorno y variables locales.  
La personalización persiste mediante los archivos de inicio.

### Scripting básico
El scripting estándar se basa en la sintaxis POSIX sh.  
Se utilizan estructuras como:
- Condicionales
- Bucles
- Lectura de entradas
- Sustitución de comandos  
Un guion debe incluir un *shebang* que indica el intérprete del script.  
El comportamiento de retorno (exit codes) es fundamental para ejecutar acciones basadas en éxito o fallo de comandos.

## 2. Interfaces de usuario y escritorios

### Sistema X Window
X11 es la base de la interfaz gráfica tradicional de Linux, gestionando entrada/salida, ventanas y comunicación entre programas gráficos.  
No es un entorno de escritorio; requiere un gestor de ventanas o un entorno completo.

### Entornos de escritorio
Xfce, GNOME, KDE Plasma y otros proporcionan escritorios completos, aplicaciones integradas y herramientas de configuración.  
Cada entorno puede diferir en bibliotecas (Qt vs GTK), consumo de recursos y características visuales.

### Acceso remoto a escritorios
Linux soporta múltiples protocolos:
- XDMCP  
- VNC  
- RDP  
- SPICE  
Cada uno es adecuado para diferentes necesidades de rendimiento, compatibilidad y seguridad.

## 3. Tareas administrativas

### Cuentas y grupos
La gestión de usuarios implica el uso de estructuras como `/etc/passwd`, `/etc/shadow`, `/etc/group` y `/etc/gshadow`.  
El esqueleto `/etc/skel` define plantillas para nuevos usuarios.  
Las contraseñas tienen políticas de expiración y bloqueo que afectan la seguridad.

### Automatización
Las tareas periódicas usan cron, organizado por usuario y por el sistema.  
Para tareas únicas se emplea at.  
La automatización permite ejecutar scripts, notificar eventos o realizar mantenimientos.

### Localización (i18n)
Linux maneja localizaciones mediante variables `LC_*` y `LANG`.  
La configuración incluye:
- idioma  
- formato numérico  
- moneda  
- zona horaria  
El sistema utiliza zonas horarias basadas en UTC como referencia y aplicadas mediante archivos vinculados en `/etc/localtime`.

## 4. Servicios esenciales del sistema

### Hora del sistema
El sistema mantiene dos relojes: el de hardware (RTC) y el de sistema.  
Las herramientas modernas usan systemd-timedated, además de ntpd o chrony para sincronización precisa.  
Sincronizar la hora es esencial para coherencia en logs, seguridad y sistemas distribuidos.

### Registros del sistema
Existen dos enfoques principales:
- `rsyslog`, basado en facilidades y niveles (daemon, auth, mail…)  
- `systemd-journald`, que implementa un registro binario con soporte estructurado  
La administración de logs permite filtrar, indexar, reenviar o rotar archivos de registro.

### Mail Transfer Agent (MTA)
En Linux, el correo local se envía mediante MTAs.  
Los sistemas comunes incluyen Postfix, Sendmail, Qmail y Exim.  
Los MTAs entregan correo tanto local como remoto, usando SMTP.  
Los usuarios tienen un archivo de bandeja de entrada como `/var/mail/usuario`.

### Impresión
El sistema de impresión estándar es CUPS, que actúa como servidor y gestor de trabajos.  
Integra colas, drivers y protocolos de impresión.

## 5. Fundamentos de redes

### Protocolos
El modelo TCP/IP define capas (acceso, red, transporte, aplicación).  
Linux implementa TCP, UDP, ICMP y protocolos auxiliares.

### Configuración de red
La configuración de red puede ser gestionada por herramientas modernas como NetworkManager o directamente por scripts.  
La tabla de rutas y las interfaces definen el flujo del tráfico.  

### Detección de problemas
Se analizan problemas mediante:
- conectividad  
- resolución DNS  
- rutas  
- servicios escuchando  
Estos diagnósticos son esenciales para mantenimiento.

### Cliente DNS
La resolución DNS depende del archivo resolv.conf o de systemd-resolved.  
Las consultas DNS se realizan mediante bibliotecas del sistema y herramientas de diagnóstico.

## 6. Seguridad

### Gestión de seguridad del sistema
Implica políticas de contraseñas, permisos, organización de archivos sensibles y bloqueo de cuentas.  
La administración incluye el control de permisos especiales, atributos y auditorías básicas.

### Seguridad del host
La seguridad local incluye desactivar servicios innecesarios, endurecer SSH, gestionar firewalls y aplicar configuraciones mínimas de exposición.  
La capa de TCP Wrappers complementa mecanismos más complejos para limitar acceso.

### Encriptación
La protección de datos se logra mediante cifrado de archivos, uso de GPG, hashes y cifrado de volúmenes.  
LUKS provee cifrado para particiones completas.
